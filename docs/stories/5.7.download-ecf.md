# Story 5.7: Download de Arquivos ECF

## Status

**Ready for Review**

---

## Story

**As a** CONTADOR,
**I want** fazer download dos três tipos de arquivo ECF,
**so that** eu possa revisar, transmitir ao SPED ou compartilhar.

---

## Acceptance Criteria

1. Use case `DownloadEcfFileUseCase` criado em `application/port/in/`

2. Service `DownloadEcfFileService` implementa:
   - Busca `EcfFile` por `id`
   - Valida que o arquivo pertence à empresa do CONTADOR autenticado (via `companyId`)
   - Retorna `ecfFile.content` como `byte[]` usando encoding **ISO-8859-1 (LATIN-1)**

3. Endpoint: `GET /api/v1/ecf/{ecfFileId}/download` (autenticado, requer header `X-Company-Id`)

4. Response headers obrigatórios:
   - `Content-Type: text/plain; charset=ISO-8859-1`
   - `Content-Disposition: attachment; filename="{fileName}"`
   - `Content-Length: {fileSizeBytes}`

5. Response 200 OK com o arquivo `.txt` no body

6. Response 404 se arquivo não existe

7. Response 403 se arquivo não pertence à empresa do CONTADOR autenticado

8. Funciona para os 3 tipos de arquivo (ARQUIVO_PARCIAL, IMPORTED_ECF, COMPLETE_ECF)

9. Testes de integração:
   - Download bem-sucedido de ARQUIVO_PARCIAL: headers corretos e conteúdo idêntico ao salvo
   - Download bem-sucedido de IMPORTED_ECF
   - Download bem-sucedido de COMPLETE_ECF
   - 403 ao tentar baixar arquivo de outra empresa
   - 404 para ID inexistente

### Encoding no download

O arquivo ECF foi armazenado como `String` (possivelmente com conteúdo LATIN-1). Ao serializar para download:

```java
// converter String → byte[] mantendo encoding LATIN-1
byte[] bytes = ecfFile.getContent().getBytes(StandardCharsets.ISO_8859_1);
```

**Importante:** O SPED ECF é um padrão brasileiro que usa ISO-8859-1. Manter esse encoding no download garante que ferramentas de validação (PVA) aceitem o arquivo.

### Headers do response

```java
HttpHeaders headers = new HttpHeaders();
headers.setContentType(MediaType.parseMediaType("text/plain; charset=ISO-8859-1"));
headers.setContentDispositionFormData("attachment", fileName);
headers.setContentLength(bytes.length);
return ResponseEntity.ok()
    .headers(headers)
    .body(bytes);
```

### Verificação de ownership

```java
// Verificar que ecfFile.companyId == companyId do header
if (!ecfFile.getCompanyId().equals(companyId)) {
    throw new AccessDeniedException("Arquivo não pertence à empresa informada");
}
```
