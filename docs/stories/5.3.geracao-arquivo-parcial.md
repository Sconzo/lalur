# Story 5.3: Geração do Arquivo Parcial (Um Único Botão)

## Status

**Ready for Review**

---

## Story

**As a** CONTADOR,
**I want** clicar em um único botão para gerar o Arquivo Parcial contendo todos os registros M dos Lançamentos da Parte B do ano fiscal,
**so that** eu possa revisar o arquivo antes de fazer o merge com o ECF Importado.

---

## Acceptance Criteria

1. Use case `GenerateArquivoParcialUseCase` criado em `application/port/in/`

2. Endpoint: `POST /api/v1/ecf/generate-parcial` (autenticado, requer header `X-Company-Id`)

3. Service `ArquivoParcialAssemblerService` implementa o use case com os seguintes passos:
   - **Passo 1**: Valida que existem `LancamentoParteB` ACTIVE para o `fiscalYear` e `companyId`
   - **Passo 2**: Chama `PartMGeneratorService.generateArquivoParcial(companyId, fiscalYear)` → retorna `String`
   - **Passo 3**: Cria `EcfFile` com `fileType = ARQUIVO_PARCIAL`, `fileStatus = DRAFT`, `content = string gerada`, `fileName = "Parcial_M_{fiscalYear}_{cnpj}.txt"`, `generatedAt = now()`, `generatedBy = usuário autenticado`
   - **Passo 4**: Persiste via `EcfFileRepositoryPort.saveOrReplace` (upsert por empresa+tipo+ano)
   - **Passo 5**: Se existir `COMPLETE_ECF` para esta empresa e ano com status `VALIDATED` ou `FINALIZED`, rebaixa para `DRAFT` e limpa `validationErrors`

4. DTO `GenerateArquivoParcialRequest`: `fiscalYear` (Integer, obrigatório)

5. DTO `GenerateArquivoParcialResponse`: `success` (boolean), `message` (String), `ecfFileId` (Long), `fileName` (String), `periodoCount` (Integer — número de M030 gerados), `lancamentosCount` (Integer — total de lançamentos processados)

6. Nome do arquivo: `Parcial_M_{fiscalYear}_{cnpj}.txt`

7. Response 200 OK com metadados da geração

8. Response 400 Bad Request se não existirem lançamentos: `"Nenhum Lançamento da Parte B encontrado para o ano fiscal {fiscalYear}"`

9. Response 403 Forbidden se empresa não pertence ao CONTADOR autenticado

10. Testes de integração:
    - Geração bem-sucedida com múltiplos lançamentos e períodos
    - `EcfFile.fileType = ARQUIVO_PARCIAL`
    - Arquivo salvo contém registros M030/M300/M305/M310 corretos
    - Tentativa sem lançamentos retorna 400
    - Regerar o arquivo do mesmo ano substitui o anterior (upsert) e rebaixa o COMPLETE_ECF existente para DRAFT

### Cálculo de periodoCount

```java
// Contar ocorrências de "|M030|" no conteúdo gerado
long periodoCount = Arrays.stream(content.split("\n"))
    .filter(line -> line.startsWith("|M030|"))
    .count();
```

### Rebaixar COMPLETE_ECF para DRAFT

```java
// Em ArquivoParcialAssemblerService, após saveOrReplace:
ecfFileRepositoryPort
    .findByCompanyAndFiscalYearAndType(companyId, fiscalYear, COMPLETE_ECF)
    .filter(ecf -> ecf.getFileStatus() == VALIDATED || ecf.getFileStatus() == FINALIZED)
    .ifPresent(ecf -> {
        ecf.setFileStatus(DRAFT);
        ecf.setValidationErrors(null);
        ecfFileRepositoryPort.saveOrReplace(ecf);
    });
```
