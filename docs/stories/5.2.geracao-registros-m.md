# Story 5.2: Serviço de Geração dos Registros M (Lógica Interna)

## Status

**Ready for Review**

---

## Story

**As a** desenvolvedor,
**I want** serviço interno que gere todos os registros do bloco M (M030/M300/M305/M310 para IRPJ, M030/M350/M355/M360 para CSLL, e M400/M410/M405 para Contas da Parte B) a partir dos dados cadastrados no sistema,
**so that** o Arquivo Parcial possa ser montado em uma única operação.

---

## Acceptance Criteria

1. Service `PartMGeneratorService` criado em `application/service/` com método principal:
   - `String generateArquivoParcial(Long companyId, Integer fiscalYear)`

2. **Grupo 1 — IRPJ (M030/M300/M305/M310):**
   - Fonte: `LancamentoParteB` com status ACTIVE, `anoReferencia = fiscalYear`, `tipoApuracao = IRPJ`
   - Para cada `mesReferencia` distinto, ordenado crescente: gera `|M030|{dataInicio}|{dataFim}|{codigoApuracao}|`
     - `dataInicio` = primeiro dia do mês no formato `DDMMYYYY` (ex: `01012024`)
     - `dataFim` = último dia do mês no formato `DDMMYYYY` (considerar ano bissexto)
     - `codigoApuracao` = `A01`..`A12`
   - Dentro de cada período: agrupa por `codigoEnquadramento`; lançamentos com mesma chave são somados em um único M300
   - Para cada grupo: gera `|M300|{codigoEnq}|{descricao}|{A/E}|{indicador}|{somaValores}|{historico}|`
     - `A/E`: `ADICAO` → `A`, `EXCLUSAO` → `E`
     - `indicador`: todos `CONTA_PARTE_B` → `1`, todos `CONTA_CONTABIL` → `2`, mix/`AMBOS` → `3`
     - `somaValores`: soma de `LancamentoParteB.valor` de todos os lançamentos do grupo
     - `historico`: `LancamentoParteB.descricao` do primeiro lançamento do grupo
     - `descricao`: descrição do `codigoEnquadramento` via `ParametroTributario`
   - Para cada lançamento do grupo:
     - Se `tipoRelacionamento ∈ {CONTA_PARTE_B, AMBOS}`: gera `|M305|{contaParteB.codigoConta}|{valor}|{D/C}|`
     - Se `tipoRelacionamento ∈ {CONTA_CONTABIL, AMBOS}`: gera `|M310|{planoDeContas.code}||{valor}|{D/C}|`
     - `D/C`: `ADICAO` → `D`, `EXCLUSAO` → `C`

3. **Grupo 2 — CSLL (M030/M350/M355/M360):**
   - Idêntico ao Grupo 1, mas para `tipoApuracao = CSLL` e usando M350/M355/M360
   - Gerado após o Grupo 1 no arquivo

4. **Grupo 3 — Contas da Parte B (M400/M410/M405):**
   - Para cada `ContaParteB` com lançamentos ACTIVE no `fiscalYear`: gera `|M400|{codigoConta}|{descricao}|...|`
   - Para cada `LancamentoParteB` com `contaParteBId` preenchido: gera `|M410|{codigoConta}|...|{valor}|...|`
   - Para cada `ContaParteB`: gera `|M405|{codigoConta}|{saldoAnterior}|{totalAdic}|{totalExcl}|{saldoAtual}|...|`
     - `saldoAnterior` = `ContaParteB.saldoInicial`
     - `totalAdic` = soma dos `valor` dos lançamentos com `tipoAjuste = ADICAO` da conta no ano
     - `totalExcl` = soma dos `valor` dos lançamentos com `tipoAjuste = EXCLUSAO` da conta no ano
     - `saldoAtual` = `saldoAnterior + totalAdic - totalExcl`
   - ⚠️ Campos exatos de M400/M410/M405 devem ser confirmados contra o layout oficial SPED ECF antes da implementação

5. **Ordem das seções no arquivo gerado:**
   ```
   [Grupo 1: blocos M030/M300/M305/M310 — IRPJ — um por mês]
   [Grupo 2: blocos M030/M350/M355/M360 — CSLL — um por mês]
   [Grupo 3: registros M400/M410/M405 — Contas da Parte B]
   ```

6. **Formatação de valores:** decimais com vírgula (`1234,56`), sem separador de milhares, 2 casas decimais

7. **Validações:** lança exceção se não existirem `LancamentoParteB` ACTIVE para o `fiscalYear`; ignora lançamentos INACTIVE

8. Testes unitários:
   - Geração correta de M030 para um mês com lançamentos IRPJ (datas corretas no formato DDMMYYYY)
   - `ADICAO` → `A` no M300 e `D` no M305/M310; `EXCLUSAO` → `E` e `C`
   - `tipoRelacionamento = AMBOS` gera tanto M305 quanto M310
   - `tipoRelacionamento = CONTA_CONTABIL` não gera M305
   - `tipoRelacionamento = CONTA_PARTE_B` não gera M310
   - Lançamentos IRPJ → M300/M305/M310; CSLL → M350/M355/M360
   - Formatação de valor com vírgula decimal
   - `somaValores` no M300 = soma dos `valor` dos lançamentos do grupo
   - Lançamentos INACTIVE são ignorados
   - Lançamentos de meses diferentes geram M030 separados
   - `M405.saldoAtual` = `saldoInicial + totalAdic - totalExcl` da `ContaParteB`
   - `ContaParteB` sem lançamentos no ano não gera M400/M410/M405

### Mapeamento campo a campo — referência rápida

| Registro | Campo | Origem |
|---|---|---|
| M030 | dataInicio | `01` + mesReferencia(2d) + anoReferencia |
| M030 | dataFim | último dia do mês (LocalDate.of(ano, mes, 1).lengthOfMonth()) |
| M030 | codigoApuracao | `"A"` + String.format("%02d", mes) |
| M300 | codigoEnquadramento | `LancamentoParteB.codigoEnquadramento` (confirmar se direto ou via ParametroTributario) |
| M300 | tipoAjuste | ADICAO→`A`, EXCLUSAO→`E` |
| M300 | indicador | CONTA_PARTE_B→`1`, CONTA_CONTABIL→`2`, mix/AMBOS→`3` |
| M300 | totalValor | soma de `LancamentoParteB.valor` do grupo |
| M300 | historico | `LancamentoParteB.descricao` do primeiro do grupo |
| M305 | codigoContaParteB | `ContaParteB.codigoConta` via `contaParteBId` |
| M305 | DC | ADICAO→`D`, EXCLUSAO→`C` |
| M310 | codigoContabil | `PlanoDeContas.code` via `contaContabilId` |
| M310 | campo vazio | `||` (pipes consecutivos, obrigatório no layout) |

### Formatação de valores

```java
// Formato BR: vírgula decimal, 2 casas, sem milhar
String formatValor(BigDecimal valor) {
    return String.format("%.2f", valor).replace(".", ",");
}
```

### Formato de datas DDMMYYYY

```java
// dataInicio = primeiro dia do mês
LocalDate inicio = LocalDate.of(fiscalYear, mes, 1);
String dataInicio = inicio.format(DateTimeFormatter.ofPattern("ddMMyyyy"));

// dataFim = último dia do mês (considera ano bissexto automaticamente)
LocalDate fim = inicio.withDayOfMonth(inicio.lengthOfMonth());
String dataFim = fim.format(DateTimeFormatter.ofPattern("ddMMyyyy"));
```
