# Story 5.6: Validação de Campos Obrigatórios (Arquivo Parcial e ECF Completo)

## Status

**Ready for Review**

---

## Story

**As a** desenvolvedor,
**I want** validador que verifique campos obrigatórios dos registros M conforme layout SPED,
**so that** arquivos gerados não sejam rejeitados pelo validador PVA da RFB.

---

## Acceptance Criteria

1. Service `EcfValidatorService` criado em `application/service/` com **3 métodos**, um por tipo de arquivo

2. Método `validateArquivoParcial(String content)` retorna `ValidationResult` (para `ARQUIVO_PARCIAL`):
   - Valida que existe ao menos um `|M030|`
   - Valida que cada M300/M350 tem: `codigoEnquadramento` não vazio, `totalValor` numérico, `indicador` válido (`1`, `2` ou `3`)
   - Valida que cada M305/M355 tem: `codigoContaParteB` não vazio, `valor` numérico, `D/C` válido (`D` ou `C`)
   - Valida que cada M310/M360 tem: `codigoContabil` não vazio, `valor` numérico, `D/C` válido (`D` ou `C`)
   - Valida consistência do indicador: se `indicador=1`, não deve existir M310/M360 filhos; se `indicador=2`, não deve existir M305/M355 filhos
   - Avisa (warning) se `totalValor` do M300/M350 divergir da soma dos `valor` dos filhos M305/M355

3. Método `validateImportedEcf(String content)` retorna `ValidationResult` (para `IMPORTED_ECF`):
   - Valida que `|M001|` existe
   - Valida que `|M990|` existe
   - Valida que todas as linhas iniciam e terminam com `|` (formato SPED básico)
   - Não valida conteúdo interno do bloco M em detalhes — arquivo é de sistema externo

4. Método `validateCompleteEcf(String content)` retorna `ValidationResult` (para `COMPLETE_ECF`):
   - Todas as validações do `validateArquivoParcial` para o bloco M
   - Valida que `|M001|` existe
   - Valida que `|M990|` existe e que a contagem de linhas do bloco M está correta
   - Valida que todo o conteúdo tem linhas iniciando e terminando com `|`

5. DTO `ValidationResult`: `boolean valid`, `List<String> errors`, `List<String> warnings`

6. Use case `ValidateEcfFileUseCase` criado com método `validate(Long ecfFileId, Long companyId)` — delega ao método correto conforme `EcfFile.fileType`

7. Endpoint: `POST /api/v1/ecf/{ecfFileId}/validate` (autenticado, requer header `X-Company-Id`)

8. Response 200 OK com `ValidationResult`

9. Se `valid = false`: atualiza `EcfFile.fileStatus = ERROR` e `validationErrors = JSON(errors)`

10. Se `valid = true`: atualiza `EcfFile.fileStatus = VALIDATED`

11. Funciona para os 3 tipos de arquivo (chama método correspondente ao tipo)

12. Testes:
    - Arquivo Parcial bem formado retorna `valid = true`
    - M300 com `indicador=1` e linha M310 filha gera erro de inconsistência
    - M990 com contagem errada gera erro no COMPLETE_ECF
    - IMPORTED_ECF sem `|M001|` gera erro
    - Status do EcfFile é atualizado corretamente após validação

### Parser de linhas SPED

```java
// Cada linha tem formato: |campo1|campo2|campo3|
// Ao fazer split("\\|"), o primeiro elemento é vazio (linha começa com |)
String[] parts = line.split("\\|", -1);
// parts[0] = "" (vazio)
// parts[1] = tipo de registro (M300, M305, etc.)
// parts[2] = primeiro campo de dados
// etc.
String tipoRegistro = parts[1];
```

### Rastreamento de pai-filho no parser

```java
String paiAtual = null;  // "M300" ou "M350"
List<String> filhosAtuais = new ArrayList<>();
BigDecimal totalFilhos = BigDecimal.ZERO;

for (String line : lines) {
    String tipo = extractTipo(line);
    switch (tipo) {
        case "M300", "M350" -> {
            // Validar o pai anterior antes de começar o novo
            if (paiAtual != null) validarConsistenciaPai(paiAtual, totalFilhos, ...);
            paiAtual = tipo;
            filhosAtuais.clear();
            totalFilhos = BigDecimal.ZERO;
            validarM300(line, result);
        }
        case "M305", "M355" -> {
            filhosAtuais.add(line);
            totalFilhos = totalFilhos.add(extractValor(line));
            validarM305(line, result);
        }
        // etc.
    }
}
```

### Validação de totalValor M300 vs soma dos filhos

O `totalValor` no M300 é comparado com a soma dos `valor` dos M305 filhos (warning, não erro, pois pode haver divergência intencional em dados do ECF Importado):

```java
BigDecimal totalNoM300 = extractTotalValor(m300line);
if (totalNoM300.compareTo(somaFilhos) != 0) {
    result.addWarning(String.format(
        "M300 código %s: totalValor=%s difere da soma dos filhos=%s",
        codigo, totalNoM300, somaFilhos));
}
```

### Validação de M990 count

```java
// Contar linhas do bloco M (entre M001 e M990, inclusive)
long linhasBloco = lines.stream()
    .filter(l -> isInBlocoM(l))
    .count();
long m990Declarado = extractM990Count(content);
if (linhasBloco != m990Declarado) {
    result.addError(String.format(
        "M990 declara %d linhas mas o bloco M tem %d linhas",
        m990Declarado, linhasBloco));
}
```
