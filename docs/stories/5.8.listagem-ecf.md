# Story 5.8: Listagem de Arquivos ECF

## Status

**Ready for Review**

---

## Story

**As a** CONTADOR,
**I want** visualizar os arquivos ECF gerados e importados para um ano fiscal,
**so that** eu possa acompanhar o status de cada arquivo e saber o que já foi gerado.

---

## Acceptance Criteria

1. Use case `ListEcfFilesUseCase` criado em `application/port/in/`

2. Endpoint: `GET /api/v1/ecf?fiscalYear=2024` (autenticado, requer header `X-Company-Id`)

3. DTO `EcfFileListResponse`:
   - `arquivoParcial`: `EcfFileSummary` ou `null` (único por empresa+ano devido ao unique constraint)
   - `ecfImportado`: `EcfFileSummary` ou `null`
   - `ecfCompleto`: `EcfFileSummary` ou `null`

4. DTO `EcfFileSummary`:
   - `id` (Long)
   - `fiscalYear` (Integer)
   - `fileType` (String)
   - `fileName` (String)
   - `fileSizeBytes` (Long — calculado a partir de `content.length()`)
   - `fileStatus` (String)
   - `generatedAt` (LocalDateTime)
   - `generatedBy` (String)
   - `validationErrors` (List\<String\> — desserializado do JSON, presente apenas se status ERROR, null caso contrário)
   - `sourceImportedEcfId` (Long, presente apenas se `COMPLETE_ECF`)
   - `sourceParcialFileId` (Long, presente apenas se `COMPLETE_ECF`)

5. Suporte a filtro opcional: `?fileType=ARQUIVO_PARCIAL` retorna apenas o tipo solicitado

6. CONTADOR só visualiza arquivos de sua própria empresa

7. Response 200 OK (mesmo que nenhum arquivo exista para o ano — retorna campos nulos)

8. Testes de integração:
   - Retorna os 3 campos separados por tipo (null se não gerado ainda)
   - CONTADOR não vê arquivos de outra empresa (403 ou response com campos nulos)

### Não retornar o campo `content` no response

O campo `content` pode ser vários MB. A listagem retorna apenas metadados. O download é feito via endpoint dedicado (Story 5.7).

```java
// No mapper de EcfFile → EcfFileSummary:
// NÃO incluir ecfFile.getContent() no DTO de listagem
// Calcular fileSizeBytes a partir do content mas não expô-lo:
summary.setFileSizeBytes((long) ecfFile.getContent().length());
```

### Desserialização de validationErrors

```java
// validationErrors está armazenado como JSON string: ["erro 1", "erro 2"]
// Desserializar apenas se status = ERROR
List<String> errors = null;
if (ecfFile.getFileStatus() == EcfFileStatus.ERROR
        && ecfFile.getValidationErrors() != null) {
    errors = objectMapper.readValue(
        ecfFile.getValidationErrors(),
        new TypeReference<List<String>>() {});
}
summary.setValidationErrors(errors);
```

### Response quando nenhum arquivo existe

```java
// Se não há ARQUIVO_PARCIAL para o ano, retornar null (não lançar erro):
EcfFileListResponse response = new EcfFileListResponse();
response.setArquivoParcial(null);   // não gerado ainda
response.setEcfImportado(null);     // não importado ainda
response.setEcfCompleto(null);      // não gerado ainda
// Response 200 com campos nulos é o comportamento correto
```

### Filtro por fileType

```java
// Se fileType foi informado, retornar apenas o campo correspondente:
if ("ARQUIVO_PARCIAL".equals(fileType)) {
    response.setEcfImportado(null);
    response.setEcfCompleto(null);
}
// Similar para os outros tipos
```
