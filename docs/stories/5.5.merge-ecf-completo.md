# Story 5.5: Geração do ECF Completo (Merge por Chave)

## Status

**Ready for Review**

---

## Story

**As a** CONTADOR,
**I want** gerar o ECF Completo fazendo o merge do ECF Importado com o Arquivo Parcial,
**so that** o ECF final contenha todos os dados do sistema externo com o bloco M atualizado com os Lançamentos da Parte B.

---

## Acceptance Criteria

1. Use case `GenerateCompleteEcfUseCase` criado em `application/port/in/`

2. Service `EcfMergerService` implementa o **algoritmo de merge por chave** com os seguintes passos:

   **Passo 1 — Carregar conteúdos:**
   - Busca `EcfFile` tipo `IMPORTED_ECF` → erro 400 se não existir
   - Busca `EcfFile` tipo `ARQUIVO_PARCIAL` → erro 400 se não existir
   - Usa `ecfFile.content` (string do banco, sem filesystem)

   **Passo 2 — Parsear o Arquivo Parcial em mapa indexado:**
   - Percorre o Parcial e indexa cada M300/M350 junto com seus filhos (M305/M310 ou M355/M360)
   - Chave: `{codigoApuracao}|{tipoRegistro}|{codigoEnquadramento}` (ex: `"A01|M300|6"`, `"A01|M350|6"`)
   - Valor: lista de strings — linha do M300/M350 + todas as linhas filhas imediatamente abaixo
   - Também indexa M030 do Parcial: `Set<String> periodosParcial` (ex: `{"A01", "A03"}`)

   **Passo 3 — Construir resultado linha a linha (granularidade M300/M350):**
   - Percorre o ECF Importado linha a linha:
     - Linhas fora do bloco M (antes de `|M001|` e após `|M990|`) → copiadas sem alteração
     - `|M001|` e `|M010|` → copiados do Importado sem alteração
     - `|M030|...|{codigoApuracao}|` → copiada do Importado; atualiza variável "período atual"
     - `|M300|{codigoEnq}|...`ou `|M350|{codigoEnq}|...`:
       - Constrói chave `"{período atual}|M300|{codigoEnq}"` ou `"|M350|..."` conforme o caso
       - Se chave **existe no Parcial** → emite o registro do Parcial + todos seus filhos; pula filhos do Importado
       - Se chave **não existe no Parcial** → copia registro do Importado e todos seus filhos sem alteração
     - `|M990|` → placeholder; recalculado no Passo 5

   **Passo 4 — Adicionar do Parcial o que não existia no Importado:**
   - Para cada M030 do Parcial não presente no Importado: adiciona o M030 inteiro (M030 + todos M300/M305/M310 + M350/M355/M360 filhos) antes do M990
   - Para cada M300/M350 do Parcial cujo codigoEnquadramento não existia em um período já presente no Importado: adiciona dentro do bloco do período correspondente no resultado

   **Passo 5 — Recalcular totalizadores:**
   - Para M300/M350 substituídos pelo Parcial: recalcula `totalValor` = soma dos `valor` dos M305/M355 filhos
   - Para M300/M350 preservados do Importado: mantém `totalValor` original
   - `|M990|{totalLinhas}|` = contagem total de todas as linhas do bloco M no resultado (incluindo M001 e M990)

   **Passo 6 — Salvar:**
   - Cria `EcfFile` com `fileType = COMPLETE_ECF`, `fileStatus = DRAFT`, `content = string resultante`, `fileName = "ECF_Completo_{fiscalYear}_{cnpj}.txt"`, `sourceImportedEcf` e `sourceParcialFile` preenchidos
   - Persiste via `EcfFileRepositoryPort.saveOrReplace`

3. Endpoint: `POST /api/v1/ecf/generate-completo` (autenticado, requer header `X-Company-Id`)

4. DTO `GenerateCompleteEcfRequest`: `fiscalYear` (Integer, obrigatório)

5. DTO `GenerateCompleteEcfResponse`: `success` (boolean), `message` (String), `ecfFileId` (Long), `fileName` (String), `fileSizeBytes` (Long), `sourceImportedEcfId` (Long), `sourceParcialFileId` (Long), `totalLinhas` (Integer)

6. Response 400 se ECF Importado não existe: `"ECF Importado não encontrado. Faça upload do arquivo ECF antes de gerar o ECF Completo."`

7. Response 400 se Arquivo Parcial não existe: `"Arquivo Parcial não encontrado. Gere o Arquivo Parcial antes de gerar o ECF Completo."`

8. Response 403 Forbidden se empresa não pertence ao CONTADOR autenticado

9. Testes de integração:
    - M300 com código presente no Parcial é substituído no resultado
    - M300 com código ausente no Parcial é preservado do Importado
    - M030 do Parcial não presente no Importado é adicionado ao final do bloco M
    - `|M010|` do Importado é sempre preservado
    - Conteúdo fora do bloco M (antes de M001 e após M990) é preservado intacto
    - `totalValor` do M300 recalculado corretamente nos registros substituídos
    - `M990` contém contagem total correta
    - `EcfFile.fileType = COMPLETE_ECF`
    - Relacionamentos `sourceImportedEcf` e `sourceParcialFile` preenchidos

### Detecção de linhas de registros no loop

```java
// Identificar tipo de linha
String[] parts = line.split("\\|");
// parts[0] vazio (linha começa com |), parts[1] = tipo de registro
String tipoRegistro = parts[1]; // ex: "M300", "M030", "M305"

// Extrair codigoApuracao do M030
// |M030|01012024|31012024|A01|
String codigoApuracao = parts[4]; // índice 4 após split por |

// Extrair codigoEnquadramento do M300
// |M300|6|Provisoes...|
String codigoEnquadramento = parts[2]; // índice 2
```

### Pular filhos do Importado após substituição

```java
boolean substituindo = false;
for (String line : importedLines) {
    String tipo = extractTipo(line); // M030, M300, M305, etc.

    if (tipo.equals("M300") || tipo.equals("M350")) {
        String chave = buildChave(periodoAtual, tipo, extractCodigo(line));
        if (parcialIndex.containsKey(chave)) {
            // Emite do Parcial
            resultado.addAll(parcialIndex.get(chave));
            substituindo = true;
        } else {
            substituindo = false;
            resultado.add(line); // preserva do Importado
        }
    } else if (tipo.equals("M305") || tipo.equals("M310") || tipo.equals("M355") || tipo.equals("M360")) {
        if (!substituindo) {
            resultado.add(line); // preserva filhos do Importado apenas se não substituindo
        }
        // Se substituindo, pula — já foram emitidos pelo Parcial
    } else {
        substituindo = false;
        resultado.add(line);
    }
}
```
